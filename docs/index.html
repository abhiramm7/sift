Keep track of the papers you read,

Sift is a fast, native macOS app for engineers and researchers to collect → tag → rate → recall papers, books, and reports. Your papers live as plain files in a folder you control, with no account to make and nothing to subscribe to.

Reference managers got heavy. Sift didn't.

Most working researchers don't need a citation engine day to day. They need to collect a paper the moment they find it, tag it so it's findable, rate it so the good stuff floats up, and recall it months later. That's the whole loop.

  • 📄

    It's yours, as plain files

    Every paper is a PDF and a small JSON file in a folder you pick. Nothing proprietary, nothing to export your way out of later. Open it in Finder, back it up, grep it, or sync it through iCloud. It's just files.

  • ⚡

    Fast and native

    A SwiftUI Mac app that launches instantly and stays out of the way. It opens PDFs in Preview instead of shipping a worse reader, so it stays small and quick.

  • 🔒

    No account, no tracking

    There's no login and no telemetry. You download it once and it's yours. It stays a tool, not a service keeping tabs on you.

  • ✨

    Smart only if you want

    If you have Claude Code or Ollama installed, Sift uses them to tag and summarize papers on your own machine. If you don't, everything else works the same. It's a convenience, not a requirement.

Sync is just iCloud Drive

Sift keeps your whole library in one folder. Put it in iCloud Drive and macOS keeps every paper, tag, and rating in sync across your Macs. There's no sync server behind Sift, and no account to set up.

The part that saves you money

Zotero gives you 300 MB of file storage free, then charges roughly $20/yr for 2 GB, $60/yr for 6 GB, or $120/yr for unlimited, and a folder of PDFs passes that quickly. Sift runs no storage service at all, so your papers sit in the iCloud storage you already pay for. The free 5 GB tier is already about 16× Zotero's free limit, and if you already pay for iCloud+ for your photos and devices, your research library rides along on it.

If you already pay for the cloud, you don't also pay for the reference manager's cloud.

Honest caveats

  • —
    Not instant push. iCloud Drive file sync carries a delay of a few seconds to a few minutes, the same as any other file in Drive.
  • —
    Apple devices only. Sift itself is macOS-only. The files are visible in the iOS Files app, but there is no iPhone or iPad Sift app.
  • —
    PDFs are small. A typical paper is 1–5 MB, so even a large library is a few hundred MB against your quota.
  • —
    Local-only is fine too. On first launch, point Sift at any folder outside iCloud Drive. It works identically; it just won't sync.

Install in three steps

This build is signed ad-hoc (not notarized), so Gatekeeper warns on first launch. The right-click trick is needed once.

  1. 1

    Download the DMG

    Grab Sift-<version>.dmg from the latest release.

  2. 2

    Drag to Applications

    Open the DMG and drag Sift.app into your Applications folder.

  3. 3

    Right-click → Open

    Right-click the app, choose Open, then Open again to clear the first-launch warning.

What Sift doesn't do

Sift is a catalog, not a citation manager. If you need BibTeX export and DOI lookups, keep your citation tool. I'd rather be honest about that up front, so the right people end up using it.

  • —

    No embedded PDF reader

    Deliberately. It opens PDFs in Preview, where macOS already does annotations well.

  • —

    No citation export

    No BibTeX, RIS, or DOI lookup. Possible future optional add-on.

  • —

    No cloud account or push sync

    It uses iCloud Drive for sync, so expect a few seconds of delay, but never a login screen.
